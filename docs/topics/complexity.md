# 計算量

## 1. 概要

計算量は、入力サイズが増えたときに処理量がどのように増えるかを表す考え方。  
アルゴリズムの速さを厳密な秒数だけで判断するのではなく、**データ量に対して処理がどのように増えるか**を見るために使う。

このトピックでは、基礎として次の3つを確認する。

- O(1)
- O(n)
- O(n^2)

## 2. 基本の動き

計算量は、実行時間そのものよりも、**入力サイズに対する増え方**を見る。

例えば次のように考える。

- 配列の特定位置を1回読むだけなら O(1)
- 配列全体を1回見るなら O(n)
- 二重ループで全組み合わせを見るなら O(n^2)

同じ「配列を扱う処理」でも、ループの回し方によって増え方が変わることが重要なポイント。

## 3. 処理の流れ

今回の実装では、同じ入力配列に対して次の3種類の処理を実行する。

1. 真ん中の要素を1回だけ読む
2. 配列全体を1回走査して合計値を求める
3. 二重ループで全ての組み合わせ数を数える

処理の流れは次の通り。

1. 入力配列を決定する
2. O(1) の例として、真ん中の要素を読む
3. O(n) の例として、配列全体を1回走査する
4. O(n^2) の例として、二重ループで全組み合わせを数える
5. 操作回数と参考実行時間を表示する
6. 各計算量の増え方の違いを確認する

## 4. 計算量

### O(1)

- 時間計算量: O(1)
- 空間計算量: O(1)

配列サイズが増えても、読む位置が決まっていれば1回のアクセスで終わる。

### O(n)

- 時間計算量: O(n)
- 空間計算量: O(1)

全ての要素を1回ずつ見るため、要素数に比例して処理回数が増える。

### O(n^2)

- 時間計算量: O(n^2)
- 空間計算量: O(1)

外側のループの中で内側のループを最後まで回すため、要素数が増えると急激に処理回数が増える。

## 5. メリット / デメリット

### メリット

- 実行時間の細かな差に依存せず、処理の増え方を把握できる
- アルゴリズム同士の比較基準になる
- 将来データ量が増えたときの重さを予測しやすい

### デメリット

- 実際の実行時間をそのまま表すものではない
- 定数倍や実装詳細までは見えない
- 小さい入力では差が体感しにくい場合がある

## 6. Java実装のポイント

- O(1) の例では `values[index]` で直接アクセスする
- O(n) の例では `for` 文で全要素を1回走査する
- O(n^2) の例では二重ループを使う
- 実行時間は `System.nanoTime()` で参考値として測る
- 学習の中心は実行時間ではなく、操作回数と理論計算量に置く

## 7. コアとなる処理・重要なコードの見どころ

今回の理解の中心は、**ループ構造の違いがそのまま計算量の違いになる点**。

### O(1) の要点

```java
int index = values.length / 2;
int value = values[index];
```

配列サイズに関係なく、読む位置を決めて1回アクセスするだけ。

### O(n) の要点

```java
for (int value : values) {
    sum += value;
    operationCount++;
}
```

全要素を1回ずつ見ていくため、処理回数は要素数に比例する。

### O(n^2) の要点

```java
for (int i = 0; i < values.length; i++) {
    for (int j = 0; j < values.length; j++) {
        pairCount++;
        operationCount++;
    }
}
```

外側ループのたびに内側ループを全て回すため、要素数が増えると処理回数が急激に増える。

## 8. 実行例

### 実行コマンド例

```bash
javac -d out $(find src -name "*.java")
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic complexity --trace
```

### 入力例

```bash
[1, 2, 3, 4, 5]
```

### 出力例の要点

- O(1) の例では1回の読み取りで終わる
- O(n) の例では要素数ぶんの走査が行われる
- O(n^2) の例では二重ループで実行回数が大きく増える
- 操作回数と参考実行時間を並べて確認できる

### 大きい入力で確認したい場合

```bash
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic complexity --size 1000
```

この場合は、O(1)、O(n)、O(n^2) の増え方の違いをより体感しやすくなる。  
ただし、学習の中心はあくまで計算量の考え方と操作回数。

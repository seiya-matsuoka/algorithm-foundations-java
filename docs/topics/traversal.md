# 走査・集計の基本

## 1. 概要

走査は、配列やリストのようなデータを先頭から順番に見ていく基本操作。  
集計は、走査の途中で合計値・最大値・最小値・件数などを求める処理。

このトピックでは、 **「全件を順番に見る」** という最も基本的な処理から、複数の集計を1回の走査で同時に求められることを確認する。

## 2. 基本の動き

基本の動きはとても単純。

1. 先頭の要素から処理を始める
2. 各要素を1つずつ見る
3. 必要な変数を更新する
4. 末尾まで見終わったら結果を返す

例えば合計値を求めるなら、見た値を順に足していく。  
最大値を求めるなら、今までで一番大きい値を保持し続ける。

## 3. 処理の流れ

今回の実装では、1回の走査で以下をまとめて求めている。

- 合計値
- 最大値
- 最小値
- 平均値
- 指定値以上の件数

処理の流れは次の通り。

1. 合計値、最大値、最小値、件数用の変数を初期化する
2. 配列の先頭から末尾まで順番に走査する
3. 各要素ごとに合計値を更新する
4. 各要素が最大値・最小値候補かを判定して更新する
5. 各要素が指定値以上なら件数を増やす
6. 最後に平均値を計算する
7. 結果をまとめて返す

## 4. 計算量

- 時間計算量: O(n)
- 空間計算量: O(1)

`n` は配列の要素数。  
全ての要素を一度ずつ見るので O(n)。  
補助的に使う変数は定数個なので O(1)。

## 5. メリット / デメリット

### メリット

- 実装が単純で理解しやすい
- 多くの基本処理の土台になる
- 1回の走査で複数の集計を同時にできる

### デメリット

- 全件を見る必要があるため、要素数が増えるとその分時間がかかる
- 条件次第では、より高速な方法（二分探索やハッシュ）に劣ることがある

## 6. Java実装のポイント

- 配列は `int[]` を使う
- 走査には `for` 文を使う
- 最大値・最小値の初期値は、配列の先頭要素を使う
- 平均値は小数で扱うため `double` を使う
- 結果は `TraversalSummary` クラスにまとめて返す

## 7. コアとなる処理・重要なコードの見どころ

今回の理解の中心は、**1回のループで複数の情報を更新している点**。

```java
for (int value : values) {
    sum += value;
    if (value > max) {
        max = value;
    }
    if (value < min) {
        min = value;
    }
    if (value >= target) {
        targetCount++;
    }
}
```

ここでは、配列を1回走査するだけで複数の集計を終えている。  
この考え方は、今後のアルゴリズム学習でも非常によく出てくる。

## 8. 実行例

### 実行コマンド例

```bash
javac -d out $(find src -name "*.java")
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic traversal --trace
```

### 入力例

```bash
[5, 3, 8, 1, 4, 9, 2]
```

### 出力例の要点

- 各要素を順に見ながら、合計・最大・最小・件数が更新される
- 最後に集計結果がまとめて表示される

### 大きい入力で確認したい場合

```bash
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic traversal --size 10000 --target 500
```

この場合は、件数や参考実行時間の差を確認しやすくなる。  
ただし、学習の中心はあくまで走査の考え方と計算量。

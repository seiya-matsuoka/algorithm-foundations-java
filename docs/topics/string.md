# 文字列

## 1. 概要

文字列は、文字が順番に並んだデータ。  
Java では `String` を使って扱う。

このトピックでは、文字列の基本として以下を確認する。

- 長さの取得
- `charAt` を使った文字参照
- 1文字ずつの走査
- 特定文字の出現回数集計
- 部分文字列の切り出し
- 文字列が不変であること

## 2. 基本の動き

文字列は、 **「文字の並び」** として扱える。  
そのため、配列に近い感覚で先頭から順番に見ていくことができる。

一方で、Java の `String` は不変のため、文字列そのものを書き換えることはできない。  
文字を置き換えたい場合は、新しい文字列を作る必要がある。

つまり、文字列の基本では次を押さえる。

- `length()` で長さを取得する
- `charAt(index)` で指定位置の文字を参照する
- `substring` で一部分を切り出す
- 走査しながら件数や位置を集計できる
- 変更が必要な場合は新しい文字列を作る

## 3. 処理の流れ

今回の実装では、次の流れで文字列の基本を確認する。

1. 入力文字列を決定する
2. 対象文字を決定する
3. 文字列を先頭から末尾まで1文字ずつ走査する
4. 対象文字の出現回数を数える
5. 対象文字が最初に現れた位置を記録する
6. 先頭文字、末尾文字、文字数を取得する
7. 部分文字列を切り出す
8. 置き換えや反転を行い、新しい文字列を作る

走査の中心は、**1文字ずつ順番に見て必要な情報を更新する部分**となる。

## 4. 計算量

### 文字列走査

- 時間計算量: O(n)
- 空間計算量: O(1)

`n` は文字列の長さ。  
各文字を1回ずつ見るため O(n)。

### 部分文字列切り出し

- 時間計算量: O(k)
- 空間計算量: O(k)

`k` は切り出す部分文字列の長さ。

### 置き換え

- 時間計算量: O(n)
- 空間計算量: O(n)

文字列をそのまま変更できないため、新しい文字列を作る必要がある。

### 反転

- 時間計算量: O(n)
- 空間計算量: O(n)

末尾から先頭まで見て、新しい文字列を組み立てる。

## 5. メリット / デメリット

### メリット

- 文字データを扱う基本として分かりやすい
- 配列と似た感覚で走査できる
- `length`、`charAt`、`substring` など基本操作が揃っている

### デメリット

- `String` は不変のため、変更系処理では新しい文字列作成が必要
- 繰り返し連結や更新を行う場合は、処理コストが増えやすい
- 配列のように直接要素を書き換えることはできない

## 6. Java実装のポイント

- 文字列長は `length()` で取得する
- 指定位置の文字は `charAt(index)` で参照する
- 部分文字列は `substring(start, end)` で切り出す
- 書き換えが必要な場合は `char[]` や `StringBuilder` を使って新しい文字列を作る
- `String` は不変であることを前提に考える

## 7. コアとなる処理・重要なコードの見どころ

今回の理解の中心は、**文字列を1文字ずつ走査しながら件数と位置を集計している部分**。

```java
for (int i = 0; i < text.length(); i++) {
    char current = text.charAt(i);

    if (current == target) {
        targetCount++;

        if (firstTargetIndex == -1) {
            firstTargetIndex = i;
        }
    }
}
```

ここでは、文字列を先頭から順番に見ながら、対象文字の件数と最初に現れた位置を同時に求めている。  
この考え方は、今後の文字列処理や探索処理でも頻出となる。

もう1つ重要なのは、**String が不変であるため、置き換えでは新しい文字列を作る点**。

```java
char[] chars = text.toCharArray();
chars[index] = newChar;
return new String(chars);
```

元の文字列を直接変更するのではなく、新しい文字列を返している点を押さえる。

## 8. 実行例

### 実行コマンド例

```bash
javac -d out $(find src -name "*.java")
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic string --trace
```

### 入力例

```bash
algorithm
```

### 出力例の要点

- 文字列を index 付きで1文字ずつ表示する
- 対象文字の件数を数える
- 最初に現れる位置を表示する
- 先頭部分や中央部分の切り出し結果を表示する
- 置き換え後も元の文字列は変わらないことを確認する
- 反転した文字列を表示する

### 任意入力で確認したい場合

```bash
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic string --input datastructure --target t --trace
```

### 大きい入力で確認したい場合

```bash
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic string --size 10000 --target a
```

この場合は、長い文字列を走査するときも、考え方は変わらず先頭から順に見るだけであることを確認しやすい。

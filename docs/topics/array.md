# 配列の基本

## 1. 概要

配列は、同じ型のデータを連続して並べて保持する基本的なデータ構造。  
Java では `int[]` や `String[]` の形で扱う。

このトピックでは、配列の基本として以下を確認する。

- インデックスを使った読み取り
- 値の更新
- 途中への挿入
- 途中からの削除
- 配列が固定長であること

## 2. 基本の動き

配列は、**インデックスを使って特定位置の値を直接読むのが得意**。  
一方で、途中への挿入や削除では、後ろの要素をずらす必要がある。

つまり、配列は以下の特徴を持つ。

- 読み取りが速い
- 更新もしやすい
- 挿入・削除はずらし処理が必要
- 長さは固定で、サイズ変更時は別配列が必要になる

## 3. 処理の流れ

今回の実装では、次の順に配列の基本操作を確認する。

1. 指定インデックスの値を読む
2. 配列をコピーして、指定位置の値を更新する
3. 新しい配列を作って、途中に値を挿入する
4. 新しい配列を作って、指定位置の値を削除する

挿入や削除は、元の配列をそのまま伸縮できないため、**新しい配列を用意して要素を移し替える**流れになる。

## 4. 計算量

### 読み取り

- 時間計算量: O(1)
- 空間計算量: O(1)

### 更新

- 今回の実装: O(n)
- 理論上の単純更新: O(1)

今回の実装では学習用に元の配列を残すためコピーしているため O(n)。  
元の配列を直接更新するだけなら O(1)。

### 挿入

- 時間計算量: O(n)
- 空間計算量: O(n)

### 削除

- 時間計算量: O(n)
- 空間計算量: O(n)

## 5. メリット / デメリット

### メリット

- インデックスによる読み取りが分かりやすい
- メモリ上で連続した構造をイメージしやすい
- 多くのアルゴリズムの土台になる

### デメリット

- 固定長で扱いづらい場面がある
- 途中への挿入や削除が重い
- 要素をずらす処理が必要になる

## 6. Java実装のポイント

- `values[index]` で値を読む
- 不正なインデックスを防ぐため、範囲チェックを行う
- 配列をコピーするときは `Arrays.copyOf` を使う
- 挿入・削除では新しい配列を作り、順番に値を詰め替える

## 7. コアとなる処理・重要なコードの見どころ

配列学習で重要なのは、**途中への挿入・削除では要素をずらす必要がある**こと。

### 挿入処理の要点

```java
for (int i = 0, j = 0; i < inserted.length; i++) {
    if (i == index) {
        inserted[i] = newValue;
    } else {
        inserted[i] = values[j++];
    }
}
```

この処理では、新しい配列を用意し、挿入位置までは元の値をコピーし、挿入位置に新しい値を入れている。

### 削除処理の要点

```java
for (int i = 0, j = 0; i < values.length; i++) {
    if (i == index) {
        continue;
    }
    removed[j++] = values[i];
}
```

削除では、削除対象を飛ばしながら新しい配列へ詰め直している。

## 8. 実行例

### 実行コマンド例

```bash
javac -d out $(find src -name "*.java")
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic array --trace
```

### 入力例

```bash
[10, 20, 30, 40, 50]
```

### 出力例の要点

- 真ん中の値を直接読み取る
- 真ん中の値を更新した配列を表示する
- 真ん中に値を挿入した配列を表示する
- 真ん中を削除した配列を表示する

### 大きい入力で確認したい場合

```bash
java -cp out io.github.seiya_matsuoka.algorithmfoundations.App --topic array --size 10000 --target 777
```

この場合は、挿入や削除で新しい配列を作る必要があることを、より大きいデータサイズで体感しやすくなる。
